**A few things I learned building these:**

1. Scope blocks for error handling are non-negotiable. The built-in "configure run after" approach gets messy fast on any flow with more than 3 actions.

2. Generic connection references make templates actually portable. Hard-coded connections mean the person importing has to rebuild half the flow anyway.

3. Construction teams don't want dashboards that require training. Yellow cells for input, everything else locked. If someone has to read a manual to fill in a daily report, they won't use it.

Happy to answer questions about the flows or how I structured them. Also open to feedback — if there's a common construction workflow I'm missing, I'd like to know.

**Title:** Released a set of construction-focused Power Automate templates with Scope-based error handling

**Body:**

Sharing a set of resources I put together after 3 years of building Power Automate flows for construction companies.

The core product is a **template pack with 10 workflows** ($49): expense claim approval, purchase order generation, daily site reports, timesheet reminders, safety incident reporting, equipment maintenance tracking, RFI processing, variation requests, subcontractor payment tracking, and project completion checklists.

Every flow uses Scope blocks for error handling (try/catch pattern) and generic connection references so they import cleanly into any environment.

Link: https://flowtechadvisory.gumroad.com/l/pa-template-pack

I also wrote a **Beginner's Blueprint** ($39) — a 45-page course that goes from triggers and actions through to deploying with Solutions and ALM. Covers expression syntax, error handling patterns, and the stuff most tutorials skip.

Link: https://flowtechadvisory.gumroad.com/l/pa-blueprint

There's a bundle with everything (including Excel templates for budgets and daily runsheets) for $99: https://flowtechadvisory.gumroad.com/l/complete-bundle

Technical details on the error handling approach: every flow wraps its main logic in a Scope action. A parallel "Catch" scope runs on failure using "configure run after" set to Failed/Skipped/TimedOut. This gives you centralized error notifications without duplicating error handling across every branch.

Connection references are set up as generic placeholders — when you import the solution, Power Automate prompts you to map them to your own connections. No manual rewiring needed.

Questions welcome. Interested in hearing if anyone has a different approach to error handling in production flows.
